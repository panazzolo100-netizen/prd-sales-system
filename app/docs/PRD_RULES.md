# PRD CRM
## Manual de Arquitetura do Sistema

Versão: 1.0

---

# Objetivo

Construir o melhor CRM para empresas de engenharia do Brasil.

O sistema deverá eliminar planilhas, WhatsApp desorganizado, retrabalho e perda de informações.

Toda funcionalidade desenvolvida deverá economizar tempo da equipe.

Se não economizar tempo, ela não será implementada.

# Regras Gerais

Nunca apagar informações.

Toda alteração gera histórico.

Todo cliente possui uma timeline.

Toda venda gera um projeto automaticamente.

Todo projeto pertence a um cliente.

Toda tarefa possui um responsável.

Toda oportunidade possui um responsável.

Toda oportunidade possui uma próxima ação.

Nenhuma oportunidade pode ficar parada sem atividade.

---

# Experiência do Usuário

Todo processo deve exigir o menor número possível de cliques.

Nenhuma tela deve confundir o usuário.

Todo botão deve ter uma função clara.

Todo formulário deve ser simples.

O sistema deve funcionar perfeitamente em notebook e celular.
